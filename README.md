# 🚪 AIアプリ開発 サバイバルキット

> **AIで初めてアプリを作る人が「致命傷を負うポイント」だけに絞った生存マニュアルです。**
> これは完成品ではなく、**実際に詰まった人の声で育てる"測定器"** です。読んだら→詰まった所を教えてください（下の「フィードバック」へ）。

## 🚨 今まさに画面が燃えている人へ（最優先）

> **エラーで真っ白／インストールが赤い文字で止まった／本番だけ動かない**
>
> → [`02_緊急_生存辞典`](./02_survival_guide.md) の **「症状→診断テーブル」** を開く。**まずログ。推理はその後。**
> → それでも無理なら、エラー全文を持って [🆘 エラー持ち込み](../../issues/new?template=02_bring_error.yml) へ。**一緒に直します。**

## 🧭 症状別ルーティング（どれを開けばいい？）

| こんな状態 | 開く冊子 |
|---|---|
| 「黒い画面（ターミナル）って何？ ログってどこ？」 | [`01_操作の地図`](./01_operation_map.md)（戦う前の"立ち位置"） |
| 「エラーで死んだ／本番で動かない／データが消える」 | [`02_緊急_生存辞典`](./02_survival_guide.md)（致命傷の応急処置） |
| 「動いてる。用語のニュアンスを落ち着いて知りたい」 | [`03_用語辞典`](./03_glossary.md)（網羅・通読用） |

> **進む順番**：① `01` で立ち位置を掴む → ② 作る/死ぬで `02` → ③ 余裕が出たら `03`。
> ただし順番通りに読む必要はない。**今の症状**が、開く冊子を決める。

## 📖 3冊の関係（重複ではなく役割分担）

- **01 操作の地図** … コマンド・画面の"手前"で固まった時。「今どこにいて、どの画面に何が映るか」。
- **02 生存辞典** … 死んだ時・死にそうな時（緊急）。致命傷の核だけに絞った版。
- **03 用語辞典** … 落ち着いて学ぶ時。用語の網羅。
- ⚠️ **01〜03で同じ用語が出たら、生存に関わる判断は `02` が正**（02の深掘りのほうが手厚い）。

## 🧪 フィードバック ｜ これは「主張」ではなく「測定器」です

この辞書が本当に効くかは、**実際に詰まった人の声でしか分かりません。** だから感想より、**失敗の生データ**を求めています。

### 一番ほしいのは「うまくいった話」ではなく「詰まった話」
> 「役に立った」報告は、**戻ってこれた人**しか書けません（＝生存者バイアス）。
> 本当に知りたいのは **「どこでまだ詰まったか」** と **「持ち込まれた生のエラー」**。だからそこを手厚く募集します。

### 入口は2つ（あなたのレベルで選んでOK）

- 🧱 **[ここで詰まった報告](../../issues/new?template=01_stuck_report.yml)** … 「役立った」より「どこで止まったか」。一番見たいやつ。
- 🆘 **[エラーを持ち込む](../../issues/new?template=02_bring_error.yml)** … 今止まってるエラーを貼って。一緒に直します。
- 🪶 **まだGitHubのIssueが怖い／操作が分からない人** … 無理にIssueを立てなくてOK。
  → **`<ここにXの募集投稿URL or Googleフォームのリンクを貼る>`** から、摩擦ゼロで聞けます。

> Issueを立てられる人は、もう Layer 0 を半分卒業しています。
> **一番届けたい"真の初心者"ほどIssueを立てられない** ので、上の摩擦ゼロ窓口を必ず1つ用意してください。

詳しい運営方針・研究フェーズの期限は [CONTRIBUTING.md](./CONTRIBUTING.md) へ。
