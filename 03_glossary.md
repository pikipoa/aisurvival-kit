> 🧭 **ここは用語をじっくり引く版。** 緊急で死んでるなら→`生存辞典 v2`｜操作画面が分からない→`Layer 0`｜全体の入口→`README`

> 【NEW】【改訂】が今回の追加・修正点。既存項目は原則そのまま活かしています。
>
> **大前提のひとこと**：「ローカルで全部作ってからGitHubに引っ越す」のではなく、**最初からGitHubに置きながら作る**のが安全。下の「フェーズ2」は時系列の後半という意味ではなく、"管理の仕組み"として最初から並走させるものだと思ってください（巨大に育ててから引っ越すと、.envの流出など事故が起きやすい）。

## 🎨 フェーズ1：AIでアプリを作る段階（生成期）

### プロンプトエンジニアリング
- **意味**：AIへの「指示書」の書き方。
- **噛み砕き**：料理で言う『味見しながら調味料を微調整する』行為。最初のレシピ（指示）が雑だと、AIは必ず焦げた料理を出す。

### システムプロンプト / ユーザープロンプト 【NEW】
- **意味**：AIの「人格・前提ルール」を決める指示（システム）と、その場の「お願い」（ユーザー）。
- **噛み砕き**：システムプロンプトは『就業規則』、ユーザープロンプトは『今日の業務指示』。Claude Code の `CLAUDE.md` は前者にあたる。ルールを先に固めておくと、毎回同じ説明を繰り返さずに済む。

### トークン (Token)
- **意味**：AIが文字を認識・計算する際の最小単位。
- **噛み砕き**：AI界の通貨。日本語は英語より手数料（トークン消費）が高くつきやすいため、調子に乗って長文を投げすぎるとすぐ制限に引っかかる。

### コンテキストウィンドウ
- **意味**：AIが一度に「記憶」できる会話の量（最大トークン数）。
- **噛み砕き**：AIの机の広さ。机が狭いと、昔の会話を押し出して忘れてしまい『え？それさっき言ったよ？』ってなる。

### ハルシネーション（幻覚）
- **意味**：AIが事実じゃないコードや存在しないライブラリを自信満々に創り出す現象。
- **噛み砕き**：飲み屋でベロベロの先輩が語る『俺、昔NASAで働いてたんだ』理論。もっともらしい嘘をつくので絶対にそのまま信じるな。

### RAG（検索拡張生成）
- **意味**：AIに「外部の最新マニュアル」を読み込ませてから答えさせる技術。
- **噛み砕き**：試験で教科書（外部データ）をチラ見しながら解答するズル賢いやり方。でもこれが一番正解率が高い。

### エージェント（Agent） / ツールコール
- **意味**：AIが自ら「コードを実行」したり「ファイルを操作」したりする機能。
- **噛み砕き**：指示を待つだけのアシスタントじゃなく、自分でドアを開けて外に出て行く『有能すぎる社員』。

### フレームワーク / ライブラリ（Next.js, Reactなど） 【NEW】
- **意味**：ライブラリ＝便利な道具の詰め合わせ。フレームワーク＝最初から間取りが決まっている家。
- **噛み砕き**：ライブラリは『電動ドライバー』、フレームワークは『建売の設計図』。AIに「Next.jsで」と一言添えるだけで、ゼロから木材を削る作業が消える。**何で作るかを決めずに始めると、AIが毎回違う流派で建て始めて魔境化する**。

### API / エンドポイント 【NEW】
- **意味**：プログラム同士がデータをやり取りする「窓口」。エンドポイントはその窓口の住所。
- **噛み砕き**：役所の『〇〇課の窓口』。「この書類ください（リクエスト）」と言うと「はい（レスポンス）」が返ってくる。エンドポイントは『3番窓口』みたいな番号。中の仕組みは知らなくていい、正しく頼めば物が出てくる。

### 依存関係（Dependencies） / package.json 【NEW】
- **意味**：アプリが動くために必要な「他人が作った部品リスト」と、その一覧表ファイル。
- **噛み砕き**：カレーに『ルウ』『鍋』が要るのと同じ。`package.json` は『買い物リスト』。**これが正しくないと、デプロイ先で「材料が足りません」とビルドが死ぬ。AI生成アプリが本番で動かない原因の9割はここ。**

### node_modules / npm install 【NEW】
- **意味**：買い物リスト（package.json）を見て、実際に部品をダウンロードして溜め込むフォルダと、その実行コマンド。
- **噛み砕き**：リストを持って実際にスーパーで買い込む行為。このフォルダは数万ファイルで超重いので **GitHubには上げない（`.gitignore`行き）**。代わりにリストだけ渡して、デプロイ先に「各自で買ってきて」と言う。

### ランタイム（Node.jsなど） / バージョン 【NEW】
- **意味**：書いたコードを実際に動かす「エンジン」。そのエンジンの型番がバージョン。
- **噛み砕き**：料理する『コンロ』。自分の部屋（localhost）がガスコンロ（Node v20）でも、デプロイ先がIH（v18）だと同じレシピで火が通らず「俺の環境では動いた」が発動する。**バージョンを揃えるのが平和への第一歩。**

## 📦 フェーズ2：GitHubで管理する仕組み（管理期）

### クローン (Clone)
- **意味**：GitHub上にあるリポジトリ（データ）を、自分のPCに丸ごとダウンロードすること。
- **噛み砕き**：お店に飾ってあるプラモデルを、自分の作業机（PC）に箱ごと持ってくること。ここからすべてが始まる。

### フォーク (Fork) 【NEW】
- **意味**：他人のリポジトリを「自分のアカウント上にまるごと複製」して、好きに改造できるようにすること。
- **噛み砕き**：Clone（自分のPCに持ってくる）と違い、Forkは『GitHub上に自分用のコピー店舗を建てる』。他人の作品を土台に改造したい時のスタート地点。

### リポジトリ（Repo）
- **意味**：コードの入れ物（保存フォルダ＋変更履歴）。
- **噛み砕き**：漫画の全巻を収納する『専用の本棚』。ただ置くだけじゃなく、『いつ・誰が・なぜ』変えたかまで記録される。

### README 【NEW】
- **意味**：リポジトリのトップに表示される「説明書・看板」。
- **噛み砕き**：店の『入口の看板』。何のアプリで、どう動かすかを書く。**3ヶ月後の自分は他人**なので、未来の自分とAIへの手紙でもある。

### ステージング（git add） 【NEW】
- **意味**：編集した変更の中から「今回セーブする分」だけを選んでカゴに入れる作業。
- **噛み砕き**：レジ（commit）に行く前に「これは買う、これは棚に戻す」とカゴに入れる段階。**初心者が一番「え、編集したのに保存されてない？」で混乱する隠れポイント。** 「add → commit の2段階」と覚える。

### コミット（Commit）
- **意味**：ステージングした変更に「セーブポイント」を打つこと。
- **噛み砕き**：ゲームのセーブポイントを"置く"行為。ただし置いただけでは戻れない（巻き戻しは下の「リバート/リセット」が担当）。「いつ・なぜ変えたか」のメモ（コミットメッセージ）を残すのが本体。

### リバート / リセット（取り消し） 【NEW・改訂の受け皿】
- **意味**：コミット（セーブ）を実際に過去の状態へ戻す操作。
- **噛み砕き**：コミットが『セーブ』なら、こっちが本物の『ロード』。**コミットしただけでは戻れない。この呪文を唱えて初めて時を巻き戻せる。**
  - `revert`＝「なかったことにする」操作自体を履歴に**残して**戻す（安全・共有向き）。
  - `reset`＝履歴**ごと消して**戻す（強力・自分のPC内向き）。

### プッシュ（Push） / プル（Pull）
- **意味**：ローカル（自分のPC）とリモート（GitHubサーバー）の送受信。
- **噛み砕き**：Push＝『提出』、Pull＝『最新のやつをもらってくる』。クラスのノートを共有する感覚。

### ブランチ（Branch） / マージ（Merge）
- **意味**：作業の分岐と合流。
- **噛み砕き**：枝分かれした川。実験的な改造は『枝（ブランチ）』でやり、ちゃんと動いたら本流（main）に『合流（マージ）』。

### プルリクエスト (Pull Request / PR)
- **意味**：自分の変更（ブランチ）を、本流（main）にマージしていいか確認を求める機能。
- **噛み砕き**：「この改造パーツ、本番に組み込んでもいいっすか？」という稟議書。個人開発でもワンクッション置くことで『やっちまった』を防げる。

### コンフリクト（Conflict）
- **意味**：同じ箇所を別々に編集して、どっちを採用するかAIじゃ決められない状態。
- **噛み砕き**：家族で『今日の夕飯』を同時に決めようとして、『カレー』と『ラーメン』で喧嘩が始まる地獄。ここは人間（あなた）が仲裁しないといけない。

### .gitignore (ギットイグノア)
- **意味**：GitHubに「アップロードしないファイル」を指定するリスト。
- **噛み砕き**：「お母さん、この引き出しは絶対見ないで！」リスト。ここにパスワードファイル（.env）を書き忘れると、全世界にクレカ情報がばら撒かれる悲劇が起きる。`node_modules` もここに入れる。

### SSHキー / アクセストークン
- **意味**：GitHubとPCを繋ぐための「合言葉」兼「鍵」。
- **噛み砕き**：マンションのオートロックを解除する『暗証番号＋指紋』。これがないと「お前、関係者じゃないだろ！」って門前払いされる。

## 🚀 フェーズ3：デプロイ（公開・運用期）※最大の心折れ地帯

### ローカルホスト (Localhost / 127.0.0.1)
- **意味**：自分のPCの中だけで動いている仮のサーバー環境。
- **噛み砕き**：『俺の部屋』。ここで完璧に動いたからといって、外の世界（デプロイ先）で動くとは限らない（エンジニアがよく言う『俺の環境では動いた』現象）。

### 開発環境 / 本番環境（dev / production） 【NEW】
- **意味**：自分のPCで試す環境と、世界に公開されている環境。設定値も挙動も別物。
- **噛み砕き**：『稽古場』と『本番の舞台』。稽古場（dev）では台本のミス（エラー詳細）が全部見えるが、本番（production）ではお客に見せないため隠される。**だから「本番でだけ出るバグ」が存在し、ログを見ないと探せない。**

### ホスティングサービス (Vercel, Renderなど)
- **意味**：作ったアプリを世界に公開するために借りるサーバー（土地）。
- **噛み砕き**：完成した家（アプリ）を建てるための『土地』。初心者はVercelやRenderといった、設定が簡単な『建売住宅』から始めると挫折しにくい。

### ビルド（Build）
- **意味**：開発用のソースコードを、本番用に「最適化＆圧縮」して固める作業。
- **噛み砕き**：材料（野菜や肉）を、お客さんに出せる『お弁当の形』に詰め直す作業。これをしないと、お客さん（ブラウザ）が食べ方を知らない。失敗の原因はだいたい「依存関係」か「バージョン違い」。

### CI/CD（自動デプロイ） 【NEW】
- **意味**：GitHubにPushしたら、自動でビルド＆公開まで一気にやってくれる仕組み。
- **噛み砕き**：Pushした瞬間に、見えない工場が勝手に弁当を作って店頭に並べてくれるベルトコンベア。便利だが **「壊れたコードをPushすると、壊れたまま全世界に即公開」される諸刃の剣**。

### 環境変数（Environment Variables / .env）
- **意味**：パスワードやAPIキーなど、「外に見せたくない設定値」を隠す場所。
- **噛み砕き**：レシピには書かない『店だけの秘密のタレ』。GitHubに上げるときは中身を空っぽにして、デプロイ先のサーバーの画面で直接タレを注ぎ込む。

### データの永続化 / データベース（DB） 【NEW】
- **意味**：ユーザーが入力したデータを「消えないように」保存する仕組みと、その保管庫。
- **噛み砕き**：**ここが最大級の罠。** Vercelなどの本番サーバーは『使い捨ての屋台』で、アプリが書き込んだファイルは再起動で全部消える。「俺の部屋では保存できたのに!」の正体。データを残したいなら、外部の倉庫（Supabaseなどのデータベース）を別途借りる必要がある。

### ポート（Port）
- **意味**：サーバー内の「通信の出入り口の番号」（例：3000, 8080）。
- **噛み砕き**：巨大マンション（サーバー）の『部屋番号』。いくら住所（IP）が合っていても、部屋番号（ポート）が違えば届かない。

### サーバーレス / 関数のタイムアウト 【NEW】
- **意味**：常時起動のサーバーを持たず、リクエストが来た時だけ一瞬動く仕組み。動ける時間に上限がある。
- **噛み砕き**：呼ばれた時だけ来る『出張シェフ』。常駐しないので安いが「10秒以内に料理を出して帰れ」という制限がある。**AIへの長い問い合わせなど重い処理が、この制限時間オーバーで突然死ぬ**ことがある。

### レート制限 / クォータ 【NEW】
- **意味**：「一定時間に使える回数の上限」。APIキーやプランごとに決まっている。
- **噛み砕き**：飲み放題の『1人◯杯まで』ルール。APIを叩きすぎると「はい、今日はもう無理」と **429エラー** で止められる。無料枠は特にケチなので、テスト中にすぐ枯れる。

### DNS（ドメインネームシステム）
- **意味**：「example.com」という文字を、実際のIPアドレス（数字の羅列）に変換する電話帳。
- **噛み砕き**：「あの人の家」に電話したい時、名前（ドメイン）を電話帳（DNS）で番号に変換する。設定後に「まだ繋がらない！」のは、この新しい電話帳が全国に配られる（浸透）のに時間がかかるから。

### SSL/TLS証明書（HTTPS化）
- **意味**：通信を暗号化する「身分証明書」付きの鍵。
- **噛み砕き**：郵便で『鍵付きの金庫』を送るイメージ。これがないと『http（鍵なし）』になり、ブラウザに『危険！』って怒られる。

### CORS（クロスオリジンリソース共有）
- **意味**：「別の場所（ドメイン）からのアクセス」を許可するか決めるルール。
- **噛み砕き**：「よそ者の入店お断り」の張り紙。フロントとバックエンドのドメインが違う場合、「お前、どこの者だ！」ってブロックされる。フロントエンド界で最も泣くポイント。

### ミドルウェア
- **意味**：アプリとOSの間に入って、認証やログなどを横やり（いい意味で）入れるソフト。
- **噛み砕き**：レストランで言う『ホールスタッフ』。厨房（アプリ）とお客さん（ブラウザ）の間に立って、「お客さん、予約ありますか？」って確認する係。

### 文字コード / UTF-8（文字化け） 【NEW】
- **意味**：文字をコンピューターが扱う数字に変換する規格。ズレると「文字化け」する。
- **噛み砕き**：**日本語ユーザー特有の地雷。**「譁・ｭ怜喧縺・」みたいな宇宙人語が出たら、これがズレている。ファイル・DB・通信のどこかで UTF-8 に揃っていないのが原因。

### キャッシュ (Cache)
- **意味**：よく使うデータを一時保存して、次回から爆速で表示する仕組み。
- **噛み砕き**：「前と同じ注文でいいっしょ？」と気を利かせる店員。コードを更新したのに画面が変わらない時は、大抵この店員が古い弁当を出し続けている（キャッシュクリアが必要）。

### ログ (Logs)
- **意味**：サーバーの裏側で記録されている「アプリの行動履歴やエラーの叫び声」。
- **噛み砕き**：『フライトレコーダー（ブラックボックス）』。画面が真っ白になって死んだ時、これを見ないとAIすら「なぜ死んだのか」推理できない。

### ステータスコード（404 / 500 ほか）
- **意味**：サーバーからの返事の種類。
- **噛み砕き**：
  - **404**＝「そのページ、ウチにないです（住所間違い）」
  - **500**＝「ウチの厨房（サーバー）がパンクして何も作れない（内部エラー）」
  - まずはこの2つでOK。**次に覚える組**として **403**（権限なし＝入店お断り）と **429**（レート制限＝飲みすぎ）も。

---

### 💡 困ったときの初心者向け切り分け順
1. **画面が真っ白／500** → まず **ログ** を見る（推理はそれから）。
2. **更新が反映されない** → **キャッシュ** クリア。
3. **ローカルでは動くのに本番で死ぬ** → **依存関係 / バージョン / 環境変数** の3点を疑う。
4. **保存したデータが消える** → **データの永続化（DB使ってる？）** を疑う。
5. **「お前どこの者だ」と弾かれる** → **CORS**。
