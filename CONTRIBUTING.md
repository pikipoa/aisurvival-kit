# CONTRIBUTING ｜ このキットの育て方

このリポジトリは「正しいマニュアルを配る」場所ではなく、**実際に詰まった人の声で辞書を測定・改良する**場所です。運営方針をここに明記します。

---

## 🕒 研究フェーズ（時間箱）

> **エラー持ち込みへの1:1伴走は、`<開始日>` から `2週間` ／ または最初の `20件` まで** を「研究フェーズ」として全力でやります。
> （日数・件数は配布時に自分で確定し、上の `< >` を埋めてください）

なぜ期限を切るか：1:1サポートは学びが最も濃い一方、**開けっぱなしにすると"無償サポートの常設"という期待が固定化**し、運営が消耗して辞書本体の更新が止まります。だから最初から箱に入れます。

研究フェーズ終了後は、持ち込みは引き続き歓迎しつつ、**回答は「辞書の該当ページを示す」形に切り替え**ます（個別対応から、辞書そのものの強化へ）。

---

## ♻️ 声を辞書へ"複利"で還す（最重要ルール）

1件の伴走を1件で終わらせない。**直したエラーは、必ず辞書の項目に変換**します。

```
詰まり報告 / エラー持ち込み
        ↓ 匿名化（個人情報・キーを除去）
        ↓ 「どの軸で詰まったか」を分類
   ┌────────────────┐
   │ 操作の手前の事故 → 01 へ追記        │
   │ 致命傷・診断の穴 → 02 のテーブル更新 │
   │ 用語のニュアンス → 03 へ追記        │
   │ DB・ローカル確認 → 別冊として新設     │
   └────────────────┘
```

> この還元ループが回らないなら、サポートはただの消耗。**「同じ詰まりが2回来たら、それは辞書の穴」** を合図に項目化してください。

### 📥 寄せられた声の扱い（透明性のために明記）

Issue・摩擦ゼロ窓口・SNSなどで寄せられた **詰まり報告やエラー内容は、個人情報やキーを除いて匿名化したうえで、この辞書本体（CC BY 4.0）に取り込むことがあります。** 報告を送ることは、この扱いへの同意とみなします。
- 取り込み時、**個人を特定できる形での氏名表示は行いません**（辞書全体のクレジットは作者に帰属）。
- 「自分の事例として名前を出してOK」な人は、その旨を添えてください。希望があれば謝辞に記載します。
- 「辞書には載せないでほしい」場合も、その旨を書いてくれれば改善の参考に留め、本体には反映しません。

---

## 🪤 生存者バイアスに注意（評価の方針）

- **「役に立った」報告は、戻ってこれた人しか書けない。** 心地よいが弱い証拠です（＝警戒信号）。
- 信用すべきは **「どこでまだ詰まったか」** と **「持ち込まれた生のエラー」**。
- だからIssueテンプレもダッシュボードKPIも、**"詰まり"と"生エラー"の数**を主指標にしてください。「いいね」や「役立った」は副指標。

---

## 🚪 フィードバックの入口は2層（単一入口の罠を踏まない）

| 層 | 相手 | 入口 |
|---|---|---|
| **卒業組** | エラー全文を貼れる／Issueを立てられる | GitHub Issue（2テンプレ） |
| **真の初心者** | 「黒い画面が分からない」段階 | 摩擦ゼロ窓口（Xリプ / Googleフォーム / DM） |

> ⚠️ **一番届けたい"真の初心者"ほど、GitHubもIssueも怖い。** Issueだけにすると、その層の声が永遠に集まらず、フィードバックが「比較的詰まってない人」に偏ります。
> `.github/ISSUE_TEMPLATE/config.yml` の `url` を、**必ず自分の摩擦ゼロ窓口に差し替えて**ください。ここが空だと二層化が機能しません。

---

## 🔬 "手前の事故"は受動的には出てこない

UI/UX以前の地雷（Nodeが入れられない、devツールを知らない、別アプリを開いていた等）は、Issueには上がってきません。本人が「何が分からないか分からない」からです。

これを掘るには、**リアルな初心者1人の画面を実況で観察**するのが最も濃い。集計されたIssueより、1人が詰まる瞬間の観察。研究フェーズ中に1〜2人、画面共有で伴走できると、辞書の"次の穴"が一気に見えます。

---

## ✍️ 編集の原則（辞書本体をいじる時）

- **想像で先に書かない。** 「あったほうがいい」で項目を増やすと、ダイエットした意味が消える。**実際の詰まりが根拠になってから**書く。
- **耐久する層だけ持つ。** 「どのボタンを押すか」は揮発するので辞書に固定しない（→スクショでAIに投げる導線へ）。
- **同じ用語が複数冊に出たら、生存判断は `02` を正とする。**
